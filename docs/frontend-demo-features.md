# Frontend Demo Features

## 本次新增演示能力

### 1. 角色差异演示

- 支持四类 mock 身份：
  - `teacher`
  - `student`
  - `admin`
  - `operator`（平台运营）
- 登录后菜单会按角色裁剪：
  - 教师：创作发起台、脚本审核台、生成队列、安全复审、视频库审计
  - 学生：创作发起台、生成队列
  - 学校管理员：审核规则管理、学校素材与标签、角色与权限
  - 平台运营：安全复审、视频库审计、审核规则管理、角色与权限、配额与成本
- 页面内按钮也按角色和按钮权限控制：
  - 教师可见：脚本确认、仅重合成配音、重绘画面并重配音、送入输出复审
  - 学生可见：被拦截任务的“修改后重新提交”
  - 学校管理员可见：学校规则说明、学校素材维护
  - 平台运营可见：新增规则、调整权重阈值、跨校巡检操作

### 2. 学校切换演示

- 头部新增“演示学校”下拉框，可在多个成都学校之间切换。
- 切换后会联动：
  - 系统名称
  - Logo 文案
  - 主题主色
  - 水印文案
  - 首页看板说明
  - 学校素材页内容
- 当前内置学校：
  - 成都石室中学
  - 成都外国语学校
  - 成都市实验外国语学校
  - 成都市泡桐树小学（天府校区）

### 3. 审核规则管理

- 新增菜单页：`审核规则管理`
- 页面内容包含：
  - 规则名称
  - 所属阶段
  - 风险等级
  - 权重
  - 规则描述
  - 命中说明
  - 维护人 / 更新时间
  - 启停状态
- 角色差异：
  - 学校管理员：查看学校侧规则与说明
  - 平台运营：查看并演示调整全局规则权重与阈值

## 关键页面变化

- 首页：按角色展示不同看板卡片，强调“菜单和看板随角色变化”
- 登录页：增加快捷演示身份按钮
- 创作发起台：按角色限制默认发起角色与说明文案
- 生成队列：按学校过滤任务，按角色显示不同操作按钮
- 安全复审：按学校过滤案例，平台运营支持跨校查看
- 视频库审计：按学校过滤结果，并补充空态
- 学校素材与标签：按学校隔离展示素材，并显示学校主题信息
- 角色与权限：新增当前角色下的菜单可见性说明

## 演示建议顺序

1. 在登录页用四个快捷身份分别登录，展示菜单变化。
2. 用头部学校切换展示系统名称、主题色和学校素材联动。
3. 切到“审核规则管理”，演示学校管理员和平台运营看到的内容不同。
4. 返回首页和生成队列，展示不同角色看到的按钮和看板差异。
