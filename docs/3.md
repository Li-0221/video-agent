# 中小学思政课程 AI 内容支持项目
## 业务方提案版

## 一、执行摘要

本项目拟建设“中小学思政课程 AI 内容支持平台”，面向中小学思政课教学、专题教育活动和区域资源建设场景，提供从主题输入、脚本生成、分镜设计、图片与视频生成，到字幕配音、内容审核、资产沉淀和发布管理的全流程能力。项目的核心目标不是建设一个通用型创作者工具，而是形成一个`可向学校、教育局和区域平台交付的教育内容生产平台`。

从业务价值看，本项目将有效解决当前学校思政资源不足、内容形式单一、视频类内容制作门槛高、更新速度慢、内容审核压力大等问题。平台通过多模型路由能力，能够在不同内容场景下动态平衡质量、稳定性和成本；通过内容审核中台和专项教育规则，能够确保文本、图片、视频、音频全链路输出符合未成年人教育场景和思政课程导向要求；通过平台化组织、权限、审计和资产管理能力，能够支撑学校和教育局级别的长期运营。

从技术与交付路径看，建议采用“`多模型生产平台 + 腾讯云审核中台 + 学校级内容资产管理平台`”的总体方案。生成层以`腾讯混元生视频`作为主生产模型，以`MiniMax-Hailuo-2.3 / 2.3-Fast / 02`作为成本优化模型，以`Kling Video 3.0 / 3.0 Omni`和`wan2.6 / wan2.7`作为精品增强模型，以`cogvideox-2`作为低成本试产模型；审核层以`腾讯云数据万象（CI）`和`腾讯云文本内容安全（TMS）`作为主审核中台，并根据业务需要预留`阿里云 AI 安全护栏`、`百度智能云司南 AI 审核`、`数美天净`等增强方案。整体建议分三期推进：一期形成 Web 平台最小可用闭环，二期扩展移动端和区域素材库，三期根据成本和信创诉求评估开源模型私有化储备。

## 二、项目背景

### 1. 政策背景

国家持续推动思政课改革创新与教育数字化转型。《新时代学校思想政治理论课改革创新实施方案》等政策均强调要创新教学方法、丰富教学资源、推动大中小学思想政治教育一体化建设。人工智能已逐步成为基础教育数字化的重要抓手，国家中小学智慧教育平台亦已将 AI 纳入教育相关场景。

因此，思政课程与 AI 技术的结合，既符合教育数字化发展方向，也具备明确政策导向和长期建设价值。

### 2. 业务背景

当前学校在思政资源建设和课堂应用中普遍面临以下问题：

1. 缺少高质量视频化、图像化资源，教师难以快速获得可直接使用的素材。
2. 传统内容偏静态和说教化，课堂吸引力和传播效果不足。
3. 视频类资源制作链路长、成本高、更新慢，难以响应专题教育和时政节点。
4. 面向未成年人传播的内容需要更高强度的审核和导向控制，而学校侧通常不具备成熟工具链。

### 3. 项目基础

本项目具备较好的落地条件：

1. 项目得到天府新区有关领导直接指导。
2. 政府相关部门可形成协同资源支持。
3. 创始团队具备多年互联网内容策划、生产和运营经验，适合做“内容 + 平台 + AI 工作流”产品。

---

## 三、项目痛点与机会

### 1. 当前痛点

1. 内容供给不足：学校很难持续获得高质量思政视频和图文素材。
2. 生产效率偏低：人工视频制作流程长、返工多、单位内容成本高。
3. 质量缺乏统一：不同学校、不同教师产出的内容标准不一致。
4. 合规压力突出：需要同时满足未成年人保护、内容导向、校园传播规范。
5. 平台能力缺失：市场上多数产品只提供模型调用，不提供学校可落地的组织权限、审核留痕、资产管理与成本分析能力。

### 2. 项目机会

1. 思政课是刚需课程，覆盖学校广、课时稳定、需求长期存在。
2. 教育数字化与 AI 工具进入学校端是趋势，市场窗口明确。
3. 项目既可做标准化平台，也可叠加定制化内容、培训与区域合作服务。
4. 如果率先构建“生成 + 审核 + 平台管理”的完整闭环，将具备较强业务壁垒。

## 四、解决方案定位

### 1. 项目定义

本项目建议定义为：

`中小学思政 AI 内容生产与审核一体化平台`

### 2. 项目目标

平台重点实现以下四类能力：

1. `内容生产能力`
   - 主题输入
   - 脚本生成
   - 分镜生成
   - 图片生成
   - 视频生成
   - 字幕与配音生成

2. `内容安全能力`
   - 文本、图片、视频、音频全链路审核
   - 思政导向规则
   - 未成年人保护规则
   - 教材一致性规则

3. `平台管理能力`
   - 学校与区域组织管理
   - 审核工作台
   - 素材资产库
   - 发布与审计

4. `运营与决策能力`
   - 模型调用成本分析
   - 审核命中情况分析
   - 资产沉淀与复用分析

### 3. 方案核心原则

1. 平台优先，而非模型优先。
2. 安全前置，而非结果补救。
3. 教育边界优先，而非创作自由优先。
4. 多模型分层调度，而非单一模型绑定。
5. 人机协同，而非纯自动化发布。

### 7. 发布与审计

支持：

1. 发布状态管理

### 1. 模型选型思路

业务方通常最关心三个维度：

1. 模型效果是否能直接用于教学内容生产。
2. 价格是否透明、平均单位成本是否可控。
3. 是否适合政企和教育场景，而不是只适合创作者社区。

因此，本项目不建议绑定单一模型，而建议采用：

1. 主生产模型
2. 成本优化模型
3. 精品增强模型
4. 低成本试产模型

2. `成本优化模型`
   - MiniMax-Hailuo-2.3
   - MiniMax-Hailuo-2.3-Fast
   - MiniMax-Hailuo-02

3. `精品增强模型`
   - Kling Video 3.0
   - Kling Video 3.0 Omni
   - wan2.6-t2v
   - wan2.6-i2v
   - wan2.7-t2v

4. `低成本试产模型`
   - cogvideox-2

### 4. 为什么不建议单一模型方案

单一模型方案的问题在于：

1. 精品内容与规模化内容无法分层，平均成本偏高。
2. 一旦模型厂商调整策略、限流或价格，交付风险会放大。
3. 不同题材内容无法进行最佳模型匹配。
4. 项目最终壁垒会退化为“某个模型的前端接入工具”。

因此，本项目真正的壁垒不在于“选中哪个模型”，而在于“平台能否对多个模型进行统一调度、统一审核和统一交付”。

## 七、内容审核与安全方案

### 1. 总体思路

内容安全不能只做“结果审核”，而必须覆盖整个生产链路。建议采用四层安全闭环：

1. 输入前审核
2. 生成过程约束
3. 输出结果审核
4. 人工复核与发布管控

### 2. 输入前审核

审核对象包括：

1. 教师输入的提示词
2. 上传的参考图片
3. 上传的参考视频
4. 上传的参考音频
5. 外部导入文本

处理原则：

1. 不合规内容直接拦截
2. 风险素材不进入生成链路
3. 重点对象进入人工复核

### 3. 生成过程约束

平台应在模型调用前，通过约束引擎补充和限制提示词，重点控制：

1. 学段边界
2. 思政导向
3. 未成年人保护
4. 历史人物与重大事件表达方式
5. 禁用娱乐化、戏谑化、低俗化表达

### 4. 输出结果审核

审核对象包括：

1. 脚本文本
2. 分镜描述
3. 字幕文本
4. 封面图
5. 视频关键帧
6. 视频音轨
7. OCR 提取文本

### 5. 人工复核

必须进入人工复核的内容包括：

1. 机审为疑似风险的内容
2. 涉及重大人物、重大历史事件的内容
3. 对外展示或重点发布的内容
4. 新模板和新专题的首批内容

## 九、预算测算说明

### 1. 测算口径说明

以下预算仅为`模型调用与审核资源预算预测`，用于业务决策和资源包预估，不含：

1. 一次性平台建设费用
2. 人员成本
3. 后续定制开发费用
4. 售后运维和驻场服务费用

### 2. 核心假设

为便于业务方理解，预算测算采用以下统一假设：

1. 以`5秒标准视频片段`作为基础测算单位。
2. 腾讯混元按`5秒720P 1.5元/条`测算。
3. 海螺按`768P 6秒 2元/条`近似测算常规内容成本。
4. 精品模型按`0.6元/秒`测算，即`5秒约3元/条`。
5. 审核预算按模型直接成本的`10%`预留，用于覆盖文本、图片、视频、音频审核及策略差异带来的浮动。
6. 若涉及精品样板内容，返工系数和人工复核成本通常更高，应单独核算。

| 标准路线     | 12 × 1.5 × 1.5 = 27 元 | 2.7 元       | 29.7 元            |
| 成本优化路线 | 12 × 2.0 × 1.5 = 36 元 | 3.6 元       | 39.6 元            |
| 精品路线     | 12 × 3.0 × 1.5 = 54 元 | 5.4 元       | 59.4 元            |

### 6. 预算结论

预算上最值得业务方关注的结论有三点：

1. `文本和图片审核成本很低，不是项目主要成本项。`
2. `视频生成成本才是主要变量，因此多模型调度是必要能力。`
3. `通过标准路线与精品路线分层，平台可以同时兼顾示范内容和规模化内容生产。`

## 十、建议的商务模式

从对外合作和业务落地角度，建议采用“平台建设 + 年度服务 + 模型资源”三段式商务模式：

### 1. 一次性平台建设费

适用于：

1. 一期平台搭建
2. 审核工作台建设
3. 权限与组织体系建设
4. 模型接入与工作流搭建
5. 基础模板中心建设

### 2. 年度平台服务费

适用于：

1. 平台维护
2. 版本迭代
3. 安全规则更新
4. 模型路由优化
5. 平台运维与培训支持

### 3. 模型与审核资源费

适用于：

1. 视频生成资源包
2. 图片与文本生成资源包
3. 内容审核资源包

### 4. 定制服务费

适用于：

1. 区域专题模板
2. 定制人物形象和内容资产库
3. 教师培训
4. 区域资源共建和示范校陪跑
